
1 - (S)ingle Responsability Principle
	"Uma classe deveria ter apenas um único motivo para mudar"
												-- Robert (Uncle Bob) Martin

2 - (O)pen Closed Principle
	"Entidades de software (classes, modulos, funções, etc.) devem estar abertas para extensão, 
	porém fechadas para modificação"			
										        -- Bertrand Meyer
										        
3 - (L)iskov Substitution Principle										        
	"Se q(x) é propriedade demonstravel dos objetos x de tipo T, 
	então q(y) deve ser verdadeiro para objetos y de tipo S, onde S é um subtipo de T."	
												-- Barbara Liskov	
												
4 - (I)nterface Segregation Principle
	"Uma classe não deveria ser forçada a depender de métodos que não utilizará."
												-- Robert (Uncle Bob) Martin												
												
5 - (D)ependency Inversion Principle
	"Abstrações não devem depender de implementações. Implementações devem depender de abstrações."
												-- Robert (Uncle Bob) Martin
